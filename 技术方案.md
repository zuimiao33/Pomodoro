# 在线 TodoList + 番茄钟技术方案（FastAPI + Vue）

## 1. 目标与范围

第一版目标是做一个可上线的 MVP：

- 用户注册/登录
- 任务管理（增删改查、完成状态）
- 番茄钟（开始、暂停、继续、完成）
- 基础统计（今日/本周专注时长）

本阶段不引入 Redis，先保证主链路稳定可用。

## 2. 技术栈

- 后端：FastAPI、SQLAlchemy 2.0、Alembic、PostgreSQL
- 前端：Vue 3、Vite、TypeScript、Vue Router、Pinia（后续可加）
- 鉴权：JWT（Access Token + Refresh Token）
- 部署：前端静态托管；后端 Docker 部署；PostgreSQL 托管

## 3. 架构设计

- 后端作为状态真相源：任务与番茄钟状态以数据库为准。
- 计时逻辑不依赖前端倒计时：
  - 保存 `duration_sec`、`elapsed_sec`、`last_started_at`
  - 刷新页面后可恢复正确剩余时间
- 接口层按当前登录用户过滤数据（`user_id`）。
- 初版同步方式采用 HTTP 拉取，后续再扩展 WebSocket。

## 4. 数据模型（MVP）

### users

- id (PK)
- email (unique)
- password_hash
- created_at

### tasks

- id (PK)
- user_id (FK -> users.id)
- title
- note
- priority (1-3)
- status (`todo` / `in_progress` / `done`)
- due_at (nullable)
- created_at
- updated_at

### pomodoro_sessions

- id (PK)
- user_id (FK -> users.id)
- task_id (FK -> tasks.id, nullable)
- mode (`focus` / `short_break` / `long_break`)
- duration_sec
- elapsed_sec
- last_started_at (nullable)
- status (`running` / `paused` / `completed` / `cancelled`)
- start_at
- end_at (nullable)
- created_at

### refresh_tokens

- id (PK)
- user_id (FK -> users.id)
- token_hash
- expires_at
- revoked_at (nullable)
- created_at

## 5. API 设计（首版）

### 鉴权

- `POST /api/v1/auth/register`
- `POST /api/v1/auth/login`
- `POST /api/v1/auth/refresh`
- `POST /api/v1/auth/logout`

### 任务

- `GET /api/v1/tasks`
- `POST /api/v1/tasks`
- `PATCH /api/v1/tasks/{task_id}`
- `DELETE /api/v1/tasks/{task_id}`

### 番茄钟

- `POST /api/v1/pomodoro/start`
- `POST /api/v1/pomodoro/pause`
- `POST /api/v1/pomodoro/resume`
- `POST /api/v1/pomodoro/finish`
- `GET /api/v1/pomodoro/current`

### 统计

- `GET /api/v1/stats/daily`
- `GET /api/v1/stats/weekly`
